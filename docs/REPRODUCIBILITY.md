# Reproducibility

This repo is designed so the benchmark can be rerun locally and audited from
checked-in inputs. Some steps call OpenAI; repeated local report runs usually
reuse cached model responses.

## What Requires OpenAI

These paths require `OPENAI_API_KEY` unless the needed cache rows already
exist locally:

- shared structured extraction in `extraction/shared_extractor.py`;
- RAG baseline verdict generation;
- RAG-ChunkLookup baseline verdict and chunk-ID generation;
- LogicPearl Track A extraction from request text.

The unit test suite does not require OpenAI. Tests mock the OpenAI call
boundary and check schemas, routing, retrieval, adjudication, and ruleset
behavior.

## What Is Cached

OpenAI extraction cache:

- `extraction/cache/openai/`

OpenAI RAG and RAG-ChunkLookup baseline cache:

- `extraction/cache/rag_baselines/`

Generated shared extraction output:

- `extraction/outputs/shared_features.100.live.openai.json`

The final report uses cached model rows when present. That makes repeated
local report runs stable, but it does not measure fresh sampling variance.

## What Is Deterministic

These steps are deterministic from checked-in inputs:

- compact FOIA authority corpus generation from `corpus/sources.toml`;
- lexical retrieval index construction;
- ruleset artifact generation from `pearl/rulesets/`;
- ruleset regression tests;
- benchmark adjudication into clean, ambiguous, and invalid buckets;
- manual clean review application;
- citation-support and trace-valid metric calculation;
- final Markdown report generation from transcripts and sidecars.

The live 100-case scenario file and split are checked in:

- `scenarios/muckrock_snapshot.100.live.json`
- `scenarios/split.100.live.json`

The approved-clean benchmark and split are also checked in:

- `scenarios/muckrock_snapshot.100.live.clean.approved.json`
- `scenarios/split.100.live.clean.approved.json`

## What Is Ignored

The following generated or local-only paths are intentionally ignored by git:

- `.venv/`
- `.pytest_cache/`
- `__pycache__/`
- `corpus/raw/`
- `rag/index.json`
- `extraction/outputs/`
- `extraction/cache/`
- `pearl/artifact/`
- `transcripts/`
- `scenarios/archive/muckrock_file_cache/`

These can be regenerated by `make final-report` except for local OpenAI cache
contents and the raw archived MuckRock file cache.

## Main Commands

Full report:

```bash
uv run make final-report
```

Offline checks:

```bash
uv run make test
uv run make regression
uv run make diff-ruleset
```

The generated final report is written to:

- `docs/qa/final-benchmark-report.md`

The trace walkthrough is written to:

- `docs/demo/trace-viewer.md`
